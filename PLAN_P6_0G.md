# P6-0G · creación del repositorio y publicación estática

1. Utilizar el repositorio público ya creado y conservar su historial limpio.
2. Extraer P6-0F.2 y situar sus contenidos directamente en la raíz del repositorio.
3. Configurar Git con la dirección `noreply` de GitHub antes del primer commit.
4. Ejecutar el bloque completo de QA y verificar `git status`.
5. Publicar la rama `main` y comprobar GitHub Actions.
6. Activar GitHub Pages desde la raíz de `main`; `index.html` será la entrada estable.
7. Ejecutar QA de la interfaz estática en Android, iPhone/iPad, Windows y macOS.
8. No desplegar directamente `servidor_ghf.py` en Internet.
9. Diseñar una API de producción independiente antes de conectar una demo pública al motor.
10. Mantener cualquier dato real exclusivamente en `LOCAL_PRIVATE`.
