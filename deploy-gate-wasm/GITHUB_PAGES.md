# Publicación en GitHub Pages · v1.4.1

## Preparación

Subir el contenido de la carpeta de la app a un repositorio. El archivo `.nojekyll` debe conservarse.

## Comprobaciones

- `index.html` carga correctamente.
- Los archivos de `assets/`, `ejemplos/` y `schemas/` se cargan por rutas relativas.
- La app abre ejemplos.
- Se puede exportar JSON e informes.
- No aparecen errores críticos en consola.

## Importante

No subir proyectos con datos personales o datos reales identificables si el repositorio es público.
