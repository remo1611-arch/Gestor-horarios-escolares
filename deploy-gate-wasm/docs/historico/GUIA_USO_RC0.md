# Guía de uso · RC0

## 1. Crear o cargar proyecto

La app permite empezar desde cero, cargar una plantilla sintética o importar un JSON exportado previamente.

Para una prueba inicial se recomienda cargar **Centro exigente sintético**.

## 2. Revisar tramos

En `Datos del centro` revise:

- nombre del tramo;
- hora de inicio;
- hora de fin;
- duración en minutos;
- tipo de tramo;
- si admite clase;
- si admite servicios.

Las guardias no se crean aquí. Aquí se crean las franjas donde después se colocan clases, DC o servicios.

## 3. Revisar profesorado

Cada docente puede tener objetivos de:

- lectivo;
- DC;
- permanencia;
- máximo total.

Estos objetivos alimentan los avisos de carga docente.

## 4. Revisar grupos y subgrupos

Los subgrupos sirven para casos como Religión/Atención educativa u otros agrupamientos simultáneos declarados.

## 5. Revisar actividades

Cada actividad debe tener:

- clasificación horaria;
- tipo;
- grupo o subgrupo si procede;
- docente responsable;
- docentes acompañantes si procede;
- espacio o zona si procede;
- sesiones semanales;
- duración;
- condiciones docentes.

## 6. Calcular

Use `Calcular horario`. Si la revisión previa detecta bloqueos, revise los avisos antes de insistir.

## 7. Revisar y ajustar

En móvil/tablet use preferentemente:

1. toque corto sobre una sesión;
2. botón `Mover aquí` en un hueco permitido.

El arrastre queda reservado para ordenador cuando el navegador lo soporte correctamente.

## 8. Documentos

Revise y exporte:

- JSON del proyecto;
- CSV por docentes, grupos o espacios;
- informe de validación;
- informe de bloqueos;
- informe de carga docente;
- HTML imprimible.

## 9. Copias

Antes de pruebas importantes, exporte JSON. El guardado local ayuda, pero el JSON es la copia recuperable más segura.
